## 当前菜单口径

渠道侧统一使用：

- 消耗明细：客户用量与计费消耗，仅商务展示。
- 交易明细：返点相关交易流水。
- 结算明细：自然月结算单与打款进度。

招商、运营不展示客户列表，也不展示“消耗明细”一级菜单。

## 推荐测试路径

1. 使用商务登录，先查看“数据概览”“我的客户”“消耗明细”“交易明细”“结算明细”。
2. 在左侧“原型操作”点击“跑通主流程”，自动完成客户绑定、上级审批、交易生成、自然月账单生成和打款归档。
3. 切换招商、运营视角，确认菜单只有“数据概览、交易明细、结算明细、我的团队、活动积分、站内信、审批中心、账号资料、规则与协议”。
4. 切换运营视角，确认“我的团队”只展示商务账号、商务消耗汇总和审批事项，不展示客户列表。
5. 切换总后台，查看“账号治理、审批中心、客户渠道归属、返点规则、活动积分池、交易明细、结算明细、BD 模型关联、BD 结算、规则与协议、审计日志”。

## 原型说明

- 原型仅使用浏览器本地状态，不接后端。
- 左侧“原型操作”和右侧“数据状态”只用于演示和测试，不属于正式后台功能。
- 数据可通过“重置演示数据”恢复初始状态。
- 结算不再包含渠道确认、BD确认、结算异议、总后台审核等旧流程；账单生成后直接进入待打款。
